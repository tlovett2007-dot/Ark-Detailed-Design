1. What This Site Is

Working documentation for a naval architecture feasibility study of Noah's Ark. Author: T. Lovett. The study applies professional naval architecture to the Genesis 6–8 account. It is not devotional literature — it is engineering analysis that takes the biblical text seriously as a primary source.

The site is published via GitHub Pages. Updates are made by editing HTML files locally and uploading via GitHub Desktop. The AI assistant writes complete updated HTML pages which the author uploads directly.

Live URL: https://tlovett2007-dot.github.io/Ark-Detailed-Design/index.html

2. Folder and File Structure

Every page lives in its own subfolder named identically to the page. The pattern is invariant:

root/
  index.html                          ← home page (root level only)
  ark-style.css                       ← shared stylesheet (root level only)
  pagename/
    pagename.html                     ← the page
    images/                           ← images for this page
    calcs/                            ← calculation files
    CAD/                              ← CAD files
    documents/                        ← reference documents
  ~site-conventions/
    ~site-conventions.html            ← this page

The tilde prefix on this folder sorts it to the top of file listings and signals it is meta, not content.

CSS path from every page: ../ark-style.css — always one level up.

Home link from every page: ../index.html

6. AI Assistant Instructions

When starting a new chat session on this project:

Base URL: https://tlovett2007-dot.github.io/Ark-Detailed-Design/ — append pagename/pagename.html for any page.

  1. Fetch this page first: https://tlovett2007-dot.github.io/Ark-Detailed-Design/~site-conventions/~site-conventions.html
  2. Fetch the index for current status of all pages: https://tlovett2007-dot.github.io/Ark-Detailed-Design/index.html
  3. Fetch the specific page being worked on before making any edits
  4. Deliver complete updated HTML files — never diffs or snippets
  5. Use dashes not underscores in all filenames
  6. Always use 0.524m cubit — never 0.518m
  7. Keep responses low verbosity — author prefers terse working style
  8. The project is in a Claude Project — search past chats for prior decisions before asking questions already answered

Working Style

  • Low verbosity. Author dislikes chatty responses.
  • Challenge formulas. Always verify sources before implementing equations.
  • No smoothing on charts. Straight segments between calculated points only — smoothing invents data.
  • XY Scatter for charts with non-uniform data. Never line charts with category axes.
  • Tone toward AiG: Respectful and generous. Corrections are engineering observations, not attacks.

8. Relationship to AiG Ark Encounter

The Ark Encounter (Answers in Genesis, Williamstown KY) uses a 0.518m Nippur cubit, moulded dimensions convention. This research uses 0.524m Royal Egyptian cubit, outside planking convention. Both describe the same vessel — the dimensional differences arise from convention, not disagreement about the Ark's size.

The Corrections page documents specific engineering errors in AiG's artistic interpretations. The tone is always respectful — AiG's broader project is valued, and corrections are offered as engineering refinements, not theological critiques.
